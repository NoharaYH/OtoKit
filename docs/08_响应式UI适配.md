# 响应式 UI 适配

## 架构红线

- `WindowSizeClass`、`DeviceTopology`、Pane 切分结果只允许存在于 `ui/` 层闭环
- `application/` 层绝不感知宽度分档、拓扑、导航呈现模式
- 业务页绝不直接依赖 `WindowSizeClass`、`DeviceTopology`、`displayFeatures` 原始值
- `ThemeExtension` 是 UI 层唯一合法的跨层渲染 DI 机制
- 响应式信息只通过 `ResponsiveLayoutScope`（InheritedWidget）下发

---

## 文件职责总览

| 文件 | 职责 |
|---|---|
| `ui/design_system/constants/layout_analyzer.dart` | 纯函数，输入宽度 + displayFeatures，输出 WindowSizeClass / DeviceTopology / hingeBounds |
| `ui/design_system/constants/responsive_layout_scope.dart` | InheritedWidget，向子树下发 availablePaneCount / primaryPaneWidth / isCompactNavigation |
| `ui/design_system/otokit_responsive_shell.dart` | 根级壳层，读取 MediaQuery，调用分析器，翻译为布局意图并渲染导航 |
| `ui/design_system/kit_navigation/kit_nav_rail.dart` | Medium+ 下的持久侧边导航组件 |
| `ui/pages/score_sync/components/score_sync_assembly.dart` | Score Sync 业务组件，消费 availablePaneCount 切换单/双栏布局 |
| `ui/pages/music_data/components/mai_music_assembly.dart` | MaiMai 曲目数据页，宽屏下限制内容区最大宽度 |
| `ui/pages/music_data/components/chu_music_assembly.dart` | Chunithm 曲目数据页，同上 |

## 布局参数

```
WindowSizeClass:
  compact   < 600dp
  medium    600–839dp
  expanded  840–1199dp
  large     ≥ 1200dp

Pane 分配（Flat 设备）:
  compact / medium  → 1 pane
  expanded / large  → 2 panes

导航模式:
  compact   → NavDeckOverlay（浮层胶囊）+ 左侧手势热区
  medium+   → KitNavRail（持久左侧 Rail）

UiSizes.getHorizontalMargin 档位:
  primaryPaneWidth > 840  → spaceXL (32dp)
  primaryPaneWidth > 600  → spaceL (24dp)
  otherwise               → screenEdgeMargin (12dp)

物理折叠屏退让阈值:
  _minPaneWidth = 280dp
```

## Phase 1：根级壳层收口

**目标**：将响应式判断从业务页中剥离，统一收口到 Shell 层。

### 核心逻辑（`otokit_responsive_shell.dart`）

```
MediaQuery → analyzeLayout() → LayoutAnalysis
                                    ↓
                         _resolvePaneCount()  →  availablePaneCount
                         primaryWidth 计算    →  primaryPaneWidth
                                    ↓
                         ResponsiveLayoutScope（InheritedWidget 下发）
                                    ↓
              isCompact ? _buildCompactLayout : _buildExpandedLayout
```

### `analyzeLayout` 纯函数（`layout_analyzer.dart`）

输入：`widthDp`、`displayFeatures`  
输出：`LayoutAnalysis { sizeClass, topology, hingeBounds }`

- 遍历 `displayFeatures` 筛选 `hinge` / `fold` 类型
- 0 个铰链 → `flat`，1 个 → `singleHinge`，≥2 个 → `dualHinge`

### `ResponsiveLayoutScope` 下发字段

| 字段 | 类型 | 说明 |
|---|---|---|
| `availablePaneCount` | `int` | 当前可用 Pane 数（1 / 2 / 3） |
| `primaryPaneWidth` | `double` | 主区域实际可用宽度（dp） |
| `isCompactNavigation` | `bool` | 是否 Compact 导航模式 |

业务页通过 `ResponsiveLayoutScope.maybeOf(context)` 消费，返回 nullable，无 Shell 环境时自动退化。

## Phase 2：Score Sync 双栏试点

**目标**：Score Sync 页在平板/宽屏下切换为难度选择器与日志面板并排的双栏布局。

### 改动文件：`score_sync_assembly.dart`

**`_buildFormView`**：外包 `Center` + `ConstrainedBox(maxWidth: 480)`，宽屏下表单不无限拉宽。

**`_buildSuccessView`**：读取 `availablePaneCount`，分两条路径：

```
availablePaneCount >= 2 → Row
  ├─ Expanded(flex: 2)  →  Column(header + difChoice)
  └─ Expanded(flex: 3)  →  SyncLogPanel（estimatedUsedHeight: 0）

availablePaneCount == 1 → Column（原有布局不变）
  ├─ header
  ├─ difChoice
  └─ Expanded → SyncLogPanel（estimatedUsedHeight: 正常值）
```

### 关键设计

- `SyncLogPanel` 的 `ValueKey('Log_${widget.gameType}')` 在两条路径下均保留，State 在布局切换时天然保活，无需额外处理
- `maybeOf` 而非 `of`，无 Shell 环境时退化为单栏
- 薄包装层 `mai_sync_page.dart`、`chu_sync_page.dart` 不动

## Phase 3：Music Data 宽度约束

**目标**：Music Data 页在宽屏下限制内容区宽度，防止列表过度拉伸。

### 改动文件

**`mai_music_assembly.dart`**：`build` 最外层包裹 `Center` + `ConstrainedBox(maxWidth: 600)`，覆盖加载态、无数据态、列表态三种路径。

**`chu_music_assembly.dart`**：在已有的 `Center` 内插入 `ConstrainedBox(maxWidth: 600)`，`Column` 占位符保持 `const`。

### 约束

- 此阶段不引入 `availablePaneCount` 分支
- Master-Detail 分栏结构预留，但不提前实施

## Phase 4：物理折叠屏接入

**目标**：为铰链设备激活 280dp 退让合并规则，防止物理分区过窄时强行切出不可用的 Pane。

### 改动文件：`otokit_responsive_shell.dart`

新增顶层私有常量与函数：

```dart
const double _minPaneWidth = 280.0;

int _resolvePaneCount(LayoutAnalysis analysis, double totalWidth)
```

### 退让规则

**Flat 设备**：沿用原逻辑（compact/medium → 1，expanded/large → 2）。

**SingleHinge**：
```
leftZone  = hinge.left
rightZone = totalWidth - hinge.right

两者均 ≥ 280dp → paneCount = 2
任一 < 280dp   → paneCount = 1（退让合并）
```

**DualHinge**：
```
zone1 = h0.left
zone2 = h1.left - h0.right
zone3 = totalWidth - h1.right

validCount = zones.where(z >= 280dp).length
paneCount  = validCount.clamp(1, 3)
```

### `primaryWidth` 修正

退让后（`paneCount == 1`）`primaryWidth` 直接返回 `totalWidth`，不再读铰链坐标：

```dart
if (paneCount == 1) {
  primaryWidth = totalWidth;                          // 退让后全宽
} else if (topology != flat && hingeBounds.isNotEmpty) {
  primaryWidth = hingeBounds.first.left;              // 物理铰链左缘
} else {
  primaryWidth = totalWidth * _defaultPrimaryRatio;   // 软比例切割
}
```

## 消费示例

业务页内读取响应式信息：

```dart
final scope = ResponsiveLayoutScope.maybeOf(context);
final paneCount = scope?.availablePaneCount ?? 1;

if (paneCount >= 2) {
  // 双栏布局
} else {
  // 单栏布局
}
```

`getHorizontalMargin` 动态读取 `primaryPaneWidth`，无需业务页手动计算边距：

```dart
// UiSizes.getHorizontalMargin(context) 内部实现
final w = ResponsiveLayoutScope.maybeOf(context)?.primaryPaneWidth ?? 0;
if (w > 840) return spaceXL;
if (w > 600) return spaceL;
return screenEdgeMargin;
```
