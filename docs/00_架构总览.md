## 二、六层架构总图

```
┌─────────────────────────────────────────────────────────┐
│                      UI 层 (ui/)                        │
│  pages/  ←  design_system/kit_*/  ←  theme/             │
│  只接收 props + callback + ThemeExtension               │
│  禁止 import get_it / dio / drift                       │
└────────────────────┬────────────────────────────────────┘
                     │ 读取状态 / 分发事件
┌────────────────────▼────────────────────────────────────┐
│               状态层 (application/)                      │
│  controllers/  states/  coordinators/                   │
│  只持有 UI State，只调用 UseCase                         │
│  通过 Stream 订阅基础设施事件                             │
└────────────────────┬────────────────────────────────────┘
                     │ 调用 UseCase
┌────────────────────▼────────────────────────────────────┐
│               业务层 (domain/)                           │
│  usecases/  entities/  value_objects/  repositories/    │
│  纯 Dart，禁止 Flutter / Drift / Dio                     │
│  通过 Repository 接口与基础设施通信                       │
└────────────────────┬────────────────────────────────────┘
                     │ 实现接口
┌────────────────────▼────────────────────────────────────┐
│             基础设施层 (infrastructure/)                 │
│  storage/  network/  native/  parsers/  games/          │
│  持有 Drift / Dio / MethodChannel / SecureStorage       │
│  对上只暴露 Repository 接口实现                           │
└─────────────────────────────────────────────────────────┘

横向支撑层（任何层均可引用）：
  shared/           # 纯 Dart：errors、result、constants、models
  bootstrap/config/ # 环境配置；仅 bootstrap 与 infrastructure 使用
  bootstrap/di/     # GetIt 注册，仅 main.dart 引用
```

## 四、依赖规则（硬性禁止）

```
❌ domain/          →  flutter / drift / dio / MethodChannel
❌ application/     →  infrastructure/（直接，须经 domain 接口）
❌ application/     →  ui/design_system/constants（依赖方向倒置）
❌ ui/kit_*         →  provider / get_it（import）
❌ ui/              →  domain/usecases（直接调用）
❌ 任何层            →  bootstrap/di（bootstrap 是装配终点，main 除外）

✅ 允许：
  ui/              → application/（读状态）、shared/
  application/     → domain/usecases、domain/repositories（类型）、shared/
  domain/          → shared/
  infrastructure/  → domain/repositories（实现接口）、shared/、bootstrap/config
```

## 五、完成标准（Definition of Done）

| 检验项 | 要求 |
|---|---|
| domain/ 无 flutter/drift/dio import | 0 处 |
| ui/design_system/ 无 provider/get_it import | 0 处 |
| application Controller 行数 | ≤ 200 行 |
| 外部交互（http/native/sql）所在层 | 仅 infrastructure/ |
| Controller 依赖注入方式 | 仅构造函数注入，禁止 getIt<>() 在构造内 |
| 主题消费方式 | ThemeExtension 单点，RootPage 单点 ValueListenableBuilder |
| 启动页偏好 | StartupBootstrap 根入口执行一次，ScoreSyncPage 不承担 |

## 六、ThemeExtension 约束

**ThemeExtension 是 UI 层唯一的跨层渲染 DI 机制。**  
组件通过 `Theme.of(context).extension<AppTheme>()` 获取主题数据，**禁止**直接 import ThemeCatalog 或 GameProvider 做主题计算。
