# 背景制作规范

> 本规范基于 maimai でらっくす CiRCLE 官网与 maimai でらっくす（初代 DX）官网的背景适配实现提炼而成。
> 所有新增或修改主题背景的 `buildBackground` 实现必须遵守本文档。

---

## 一、核心原则

**"层级分离、锚点固定、断点切档、溢出裁切"**

1. **层级分离**：背景拆为独立的功能层，每层有唯一的定位策略与职责
2. **锚点固定**：每层元素锚定在视口的几何特征点（中心、四角、边缘）上，而非随屏幕做连续比例缩放
3. **断点切档**：通过有限的宽度断点切换每层的具体尺寸参数，不使用 `screenWidth * ratio` 做连续缩放
4. **溢出裁切**：超出视口的部分由 `Stack` 的 `clipBehavior` 自然裁掉，不需要精确计算溢出

---

## 二、层级模型

每个主题的 `buildBackground` 返回 `Stack(fit: StackFit.expand)` 内部，从底到顶最多包含以下 5 层。不是每层都必须存在，但**层序固定**：

| 层序 | 名称 | 职责 | 定位策略 | 官网对应 |
|---|---|---|---|---|
| L0 | 底色层 | 纯色或渐变填充 | `Positioned.fill` | `radial-gradient` / `ColoredBox` |
| L1 | 纹理层 | 可平铺或旋转的满屏花纹 | 视口中心锚定，正方形 | `pattern-layer` / `headdot` |
| L2 | 焦点层 | 主视觉焦点（圆环、地球等大型图案） | 视口中心或边缘锚定 | `circle-*` / `BgContent(earth)` |
| L3 | 边角层 | 四角或边缘的静态装饰 | 四角锚定 | `corner.*` |
| L4 | 氛围层 | 漂浮的小装饰（星星、云朵、3D 物件） | 视口百分比定位 | `tile` / `star` / `cloud` / `threeDemention` |

---

## 三、尺寸档位

在 `buildBackground` 内部通过 `MediaQuery.sizeOf(context).width` 判定当前档位：

| 档位 | 宽度范围 | 典型设备 | 官网对应 |
|---|---|---|---|
| `compact` | < 600 dp | 手机竖屏 | `@media (max-width: 600px)` |
| `expanded` | ≥ 600 dp | 平板、桌面、手机横屏 | PC 默认档 |

### 为什么只有两档

官网只做了 PC / SP 两档。在背景这一纯装饰层面，中间态的视觉差异人眼不敏感，增加档位只增加维护成本而不会带来可感知的收益。

### 判定方式

`buildBackground` 直接读取 `MediaQuery`。背景层位于 `PageShell` 内部、`ResponsiveLayoutScope` 之上，属于 `ui/` 层内闭环，不涉及 `application/`，不违反架构红线。

```dart
final w = MediaQuery.sizeOf(context).width;
final bool isCompact = w < 600;
```

## 四、各层制作规范

### L0 底色层

**作用**：填满整个视口的纯色或渐变，作为最底层的视觉基础。

**写法固定**：

```
Positioned.fill
  └─ Container(decoration: BoxDecoration(gradient / color))
```

**要求**：
- 无任何固定像素值，天然适配所有尺寸
- 不需要分档
- 渐变方向与色值为主题品牌固定色，不受 `dark/light` token 影响

---

### L1 纹理层

**作用**：覆盖在底色之上的花纹/纹理，通常做低速旋转或平铺。

**核心思路**：以视口中心为锚点的正方形，**边长取视口长边**（`max(width, height)`），然后在该正方形上做旋转或平铺。

**Widget 结构**：

```
Positioned.fill
  └─ Center
       └─ SizedBox(width: longEdge, height: longEdge)
            └─ Transform.rotate(angle: animation)
                 └─ Image.asset(fit: BoxFit.contain)
```

**规则**：
- 容器尺寸 = `max(viewportWidth, viewportHeight)`，保证无论横竖屏花纹都无留白
- 图片填充使用 `BoxFit.contain`（保持比例铺满正方形）或 `ImageRepeat.repeat`（平铺纹理）
- 允许用 `AnimationController` + `Transform.rotate` 做低速旋转（建议 ≥ 120 秒/圈）
- **不需要分档**：因为尺寸已经是自适应逻辑

---

### L2 焦点层

**作用**：背景的主视觉焦点，如旋转的圆环、地球、彩虹等大型图案。

**核心思路**：锚定到视口几何特征点，使用**固定像素尺寸**，按档位切换。

**锚点类型**（二选一）：

| 锚点 | 定位方式 | 适用场景 |
|---|---|---|
| 中心型 | `Positioned.fill` + `Center` | 旋转圆环、居中图案 |
| 边缘型 | `Positioned(bottom/top: offset)` + 水平居中 | 地球（从底部露出）、天空元素 |

**Widget 结构**（中心型）：

```
Positioned.fill
  └─ Center
       └─ AnimatedBuilder(animation: rotateController)
            └─ Image.asset(
                 width: isCompact ? compactSize : expandedSize,
                 height: isCompact ? compactSize : expandedSize,
                 fit: BoxFit.contain,
               )
```

**尺寸规则**：

```dart
final double size = isCompact ? compactSize : expandedSize;
```

- `compactSize`：设计稿在 ~393dp 宽度下的视觉尺寸
- `expandedSize`：设计稿在 PC/平板下的目标尺寸（参考官网 PC 值或素材原始分辨率的合理比例）

**关键约束**：
- **禁止**使用 `screenWidth * ratio` 做连续缩放
- **禁止**使用裸 `Transform.scale` 充当缩放手段（应改为直接设置 `width/height`）
- **允许**元素溢出视口，被 Stack 裁掉
- 动效参数（旋转速度、方向）不分档，在所有尺寸下保持一致

---

### L3 边角层

**作用**：锚定在视口四角的静态装饰图。

**核心思路**：锚定到视口四角，使用固定像素尺寸，按档位切换。宽边装饰在 compact 下可拉满视口宽度。

**Widget 结构**：

```
Positioned(top/bottom: 0, left/right: 0)
  └─ Image.asset(
       width: isCompact ? compactWidth : expandedWidth,
       fit: BoxFit.contain,
     )
```

**尺寸规则**：

```dart
// 宽边角（横向延展的装饰）
final double width = isCompact ? double.infinity : expandedWidth;

// 窄边角（纵向延展的装饰）
final double width = isCompact ? compactWidth : expandedWidth;
```

**关键约束**：
- `compact` 下宽边角用 `double.infinity` + `BoxFit.contain`（贴边无留白）
- `expanded` 下给到接近官网 PC 的固定尺寸
- **不做动效**，边角是静态锚定物

---

### L4 氛围层

**作用**：漂浮的小装饰，营造动态氛围。

**核心思路**：使用视口百分比定位 + 固定像素尺寸 + 循环动效，位置和尺寸按档位切换。

**Widget 结构**：

```
Positioned(
  top: viewportHeight * topRatio,
  left/right: viewportWidth * sideRatio,
)
  └─ AnimatedBuilder(animation: driftController)
       └─ Image.asset(
            width: isCompact ? compactSize : expandedSize,
            fit: BoxFit.contain,
          )
```

**动效类型**（可组合）：

| 类型 | 效果 | 适用 |
|---|---|---|
| 旋转 | `Transform.rotate` 低速循环 | 3D 物件、星星 |
| 漂移 | `AnimationController` 驱动水平/垂直位移 | 云朵 |
| 浮现消隐 | `Opacity` + 位移组合 | 瓷砖、星星（对应官网 `riseAndFade`） |

**关键约束**：
- compact 下**可减少元素数量**（性能考量）
- 位置定位使用视口百分比，不使用固定像素偏移
- 元素尺寸使用固定像素两档切换

## 五、参数表组织规范

每个主题的 `buildBackground` 方法内部，**在方法顶部**集中声明所有分档参数：

```dart
@override
Widget buildBackground(BuildContext context) {
  final w = MediaQuery.sizeOf(context).width;
  final h = MediaQuery.sizeOf(context).height;
  final bool isCompact = w < 600;

  // ── L1 纹理层 ──
  final double textureSide = max(w, h);

  // ── L2 焦点层 ──
  final double circleWhiteSize   = isCompact ? 354.0 : 788.0;
  final double circleYellowSize  = isCompact ? 400.0 : 900.0;
  final double circleColorSize   = isCompact ? 360.0 : 820.0;

  // ── L3 边角层 ──
  final double topLeftWidth      = isCompact ? w     : 600.0;
  final double topRightWidth     = isCompact ? 162.0 : 316.0;
  final double bottomLeftWidth   = isCompact ? 118.0 : 231.0;
  final double bottomRightWidth  = isCompact ? w     : 600.0;

  // ── L4 氛围层 ──
  final double cloudSize         = isCompact ? 110.0 : 160.0;

  return Stack(
    fit: StackFit.expand,
    children: [
      // L0 → L1 → L2 → L3 → L4
    ],
  );
}
```

**目的**：所有与尺寸相关的决策集中在方法顶部，后续 Widget 树只引用变量，不出现裸数字或 `screenWidth * ratio`。

## 六、素材要求

| 要求 | 说明 |
|---|---|
| 最低分辨率 | 每个素材至少满足 `expanded` 档的尺寸需求（推荐宽边 ≥ 1200px） |
| 格式 | 优先 `.webp`（透明通道场景）或 `.png`（需极致清晰的小图标） |
| 命名 | 统一存放在 `assets/background/{game}/{theme}/` 下，文件名使用 `snake_case` |
| SP 替换图 | 如素材在 compact 下需要完全不同的构图，可提供单独的 SP 版本（如 `dot_bg_sp.webp`） |

## 七、新增主题 Checklist

制作新主题背景时，按以下步骤执行：

| 步骤 | 内容 | 验收标准 |
|---|---|---|
| 1. 确定层级 | 从 L0–L4 中选择需要的层，画出层级结构草案 | 草案明确标注每层内容 |
| 2. 准备素材 | 每个素材至少提供 expanded 档分辨率 | 在 1200dp 宽度下无可见模糊 |
| 3. 定义参数表 | 为每个元素填写 compact / expanded 两组固定值 | 参数表写在方法顶部 |
| 4. 搭建 Widget 树 | 按 L0 → L4 逐层搭建 | 层序与本规范一致 |
| 5. 验证 compact | 在 393dp 宽度下检查视觉 | 与设计稿一致，无明显留白或溢出 |
| 6. 验证 expanded | 在 1024dp+ 宽度下检查视觉 | 无素材模糊、无比例失调、无不自然留白 |
| 7. 验证切档过渡 | 在 600dp 附近来回调整窗口宽度 | 切档时无突兀跳变（可接受瞬间重排） |

## 八、与现有架构的关系

| 维度 | 规则 |
|---|---|
| 入口 | 每个主题通过 `AppTheme.buildBackground(BuildContext context)` 返回 Widget，入口不变 |
| 尺寸来源 | `MediaQuery.sizeOf(context)`，不依赖 `ResponsiveLayoutScope` |
| DI 机制 | `ThemeExtension` 仍是唯一跨层渲染 DI，颜色 token 由 `AppTheme` 提供 |
| `_DynamicAppTheme` | 其 `buildBackground` 代理调用 `baseTheme.buildBackground(context)`，新策略透明兼容 |
| 禁止事项 | 背景层不引入新的 InheritedWidget 或全局状态，不依赖 `application/` 层任何信息 |
